# 菜品业务模型与字段参考

> 基于 `disAddGoodsNx`（添加页）与 `disEditGoodsNx`（编辑页）分析整理  
> 项目：社区电商管理后台（小程序）

---

## 一、业务概览

这是一个**社区电商/社区团购**的菜品管理系统。每个菜品属于一个**三级分类体系**，支持 4 种售卖类型，可配置售卖时间、配送方式、促销活动、会员卡绑定和首页广告。

### 业务层级关系

```
社区 (Community)
  └── 分类-爷级 (grandId)
        └── 分类-父级 (fatherId)
              └── 商品 (Goods)
                    ├── 基本信息（名称/规格/价格/图片/视频/说明）
                    ├── 配送方式（自提/外卖）
                    ├── 售卖时间（全时段/分时段）
                    ├── 售卖类型 → 决定子商品结构
                    ├── 促销活动（划线价/满减）
                    ├── 会员卡绑定
                    └── 首页广告（限时/限量）
```

## 二、四级售卖类型与子结构

### 2.1 类型对照表

| type 值 | 名称 | 含义 | 是否有属性层 | 是否有子商品 |
|---------|------|------|:---:|:---:|
| `0` | 普通 | 单菜品直接售卖 | ❌ | ❌ |
| `1` | 自由拼 | 顾客自选子商品组合，按单品累加计价 | ❌ | ✅ |
| `2` | 套餐 | 按属性分组的固定套餐结构 | ✅ | ✅ |
| `3` | 多拼 | 固定一口价，顾客自由选择子商品 | ❌ | ✅ |

### 2.2 各类型子结构详解

#### 普通 (type=0)

```
商品 ─── 直接售卖，无子结构
```

最简单的模式，用户直接下单购买。

---

#### 自由拼 (type=1)

```
商品
  ├── 最低份数 (nxCgSetSubNumber)
  ├── 最高份数 (nxCgSetSubLimitNumber)
  └── 子商品列表 (nxCommunityGoodsSetItemEntities)
        ├── 子商品1：名称、默认份数、限制份数、额外价格
        ├── 子商品2：...
        └── ...
```

- 顾客在 [最低份数, 最高份数] 范围内，从子商品列表自由选择搭配
- 总价 = 所有被选子商品的额外价格累加

---

#### 套餐 (type=2) ⭐ 最复杂

```
商品
  └── 属性列表 (nxCommunityGoodsSetPropertyEntities)  ← 套餐特有
        ├── 属性1：名称、总份数限制
        │     ├── 子商品A（可关联到其他商品）(nxCommunityGoodsSetItemEntities)
        │     └── 子商品B
        ├── 属性2
        │     ├── 子商品C
        │     └── 子商品D
        └── ...
```

- **属性层**是套餐独有的中间分组概念（如"主食"、"饮品"、"小菜"）
- 每个属性有独立的份数限制，其下的子商品份数不能超过该限制
- 子商品可通过 `nxCgsiCgGoodsId` 关联到系统已有商品
- 子商品支持"允许替换"机制（售罄时自动用同类其他子商品替换）

---

#### 多拼 (type=3)

```
商品
  ├── 最低份数 (nxCgSetSubNumber)
  ├── 最高份数 (nxCgSetSubLimitNumber)
  └── 子商品列表 (nxCommunityGoodsSetItemEntities)
        └── 价格固定，用户任选
```

- 与自由拼类似，但**价格固定**
- UI 提示："价格固定，顾客可以随意选择子商品"

---

## 三、核心字段定义

### 3.1 商品主表 (nxCommunityGoods / nxCg*)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCommunityGoodsId` | number | 商品主键 |
| `nxCgGoodsName` | string | 商品名称（必填） |
| `nxCgGoodsStandardname` | string | 规格（必填） |
| `nxCgGoodsPrice` | number | 会员价/单价（必填） |
| `nxCgGoodsDetail` | string | 商品说明（最多1000字） |
| `nxCgGoodsSort` | number | 排序号 |
| `nxCgGoodsType` | number | 售卖类型：0=普通, 1=自由拼, 2=套餐, 3=多拼 |
| `nxCgSellType` | number | 售卖时间：0=全时段, 1=分时段 |
| `nxCgServiceType` | number | 配送方式：0=自提, 1=外卖 |
| `nxCgStartTime` | string | 售卖开始时间（分时段时有效，格式 HH:mm） |
| `nxCgStopTime` | string | 售卖结束时间 |
| `nxCgCommunityId` | number | 所属社区ID |
| `nxCgCfgGoodsFatherId` | number | 父级分类ID |
| `nxCgDfgGoodsGrandId` | number | 爷级分类ID |
| `nxCgCommerceId` | number | 商户ID |
| `nxCgGoodsStatus` | number | 今日售罄：0=正常, -1=售罄 |
| `nxCgPullOff` | number | 下架：0=在售, 1=已下架 |
| `nxCgPurchaseAuto` | number | 自动采购标记（默认 -1） |

**商品图片/视频：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCgNxGoodsFilePath` | string | 商品主图路径 |
| `nxCgNxGoodsTopFilePath` | string | 顶部图路径（展示在商品详情上方） |
| `nxCgGoodsIntroVideo` | string | 介绍视频路径 |

**净菜重量（`nxCommunityType == 1` 时显示）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCgGoodsGrossWeight` | number | 毛重（斤），输入时自动算毛单价 |
| `nxCgGoodsGrossPrice` | number | 毛单价（元/斤） |
| `nxCgGoodsNetWeight` | number | 净重（斤），输入时自动算净单价 |
| `nxCgGoodsNetPrice` | number | 净单价（元/斤） |

**自由拼/多拼份数设置：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCgSetSubNumber` | number | 最少需选份数（自由拼/多拼） |
| `nxCgSetSubLimitNumber` | number | 最多可选份数（自由拼/多拼/子商品上限） |

**子商品/属性列表（嵌套返回）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCommunityGoodsSetItemEntities` | array | 子商品列表（自由拼/套餐/多拼） |
| `nxCommunityGoodsSetPropertyEntities` | array | 套餐属性列表（仅套餐） |

---

### 3.2 套餐属性 (SetProperty / nxCgsp*)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCommunityGoodsSetPropertyId` | number | 属性主键 |
| `nxCgspName` | string | 属性名称，如"主食"、"饮品"（必填） |
| `nxCgspCgGoodsId` | number | 所属商品ID |
| `nxCgspSort` | number | 排序号 |
| `nxCgspLimitAmount` | number | 该属性下总份数限制（必填） |
| `nxCommunityGoodsSetItemEntities` | array | 该属性下的子商品列表（嵌套返回） |

**约束规则：**子商品的份数和限制份数都不能超过其所属属性的 `nxCgspLimitAmount`。

---

### 3.3 子商品 (SetItem / nxCgsi*)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCommunityGoodsSetItemId` | number | 子商品主键 |
| `nxCgsiItemName` | string | 子商品名称（必填） |
| `nxCgsiItemQuantity` | number | 默认选择份数（必填） |
| `nxCgsiItemLimitCount` | number | 限制最高份数（必填） |
| `nxCgsiItemPrice` | number | 额外价格/子商品价格（必填） |
| `nxCgsiItemHuaxianPrice` | number | 子商品划线价 |
| `nxCgsiItemType` | number | 允许替换：0=不允许, 1=允许 |
| `nxCgsiItemStatus` | number | 今日售罄：0=正常, -1=售罄 |
| `nxCgsiItemSort` | number | 排序号 |
| `nxCgsiItemFilePath` | string | 子商品图片路径 |
| `nxCgsiCgPropertyId` | string | 所属属性ID（套餐时非空，自由拼/多拼时为空字符串） |
| `nxCgsiItemCgGoodsId` | number | 关联的商品ID（可指向系统已有商品） |
| `nxCommunityGoodsEntity` | object | 关联商品的详情（嵌套返回，含 `nxCgGoodsName` 等） |

---

### 3.4 促销活动 (Promotion)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCgPromotionType` | number | 促销类型：0=无, 1=划线价, 2=满减 |
| `nxCgGoodsHuaxianPrice` | number | 划线价（原价/划线价格） |
| `nxCgGoodsHuaxianQuantity` | number | 划线价每日限量份数 |
| `nxCgGoodsHuaxianPriceDifferent` | number | 划线价差价（前端自动计算：划线价-单价） |
| `nxCgPromotionPrice` | number | 满减优惠价格 |
| `nxCgPromotionAmount` | number | 满减份数门槛 |
| `nxCgPromotionWords` | string | 活动说明文字 |

**促销验证规则：**
- 划线价：`nxCgGoodsHuaxianPrice` 和 `nxCgGoodsHuaxianQuantity` 必须同时填写，且划线价 > 商品单价
- 满减：`nxCgPromotionPrice` 和 `nxCgPromotionAmount` 必须同时填写
- 切换促销类型会清空上一个类型的字段

---

### 3.5 会员卡绑定 (Card)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCgCardId` | number | 绑定的会员卡ID |
| `nxCommunityCardEntity` | object | 会员卡详情（嵌套返回，含 `nxCommunityCardId`, `nxCommunityCardName`） |

---

### 3.6 首页广告 (Adsense)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `nxCgIsOpenAdsense` | number | 是否开启广告：0=关闭, 1=开启 |
| `nxCgAdsenseStartTime` | string | 广告开始时间（HH:mm） |
| `nxCgAdsenseStopTime` | string | 广告结束时间（HH:mm） |
| `nxCgAdsenseStockQuantity` | number | 广告限量总份数 |
| `nxCgAdsenseRestQuantity` | number | 广告剩余份数 |

## 四、创建/编辑页面关键差异

| 维度 | 添加页 (disAddGoodsNx) | 编辑页 (disEditGoodsNx) |
|------|------------------------|---------------------------|
| 数据来源 | 用户输入，构建 data 对象 | `comGetGoodsDetail` 接口获取 `goods` 对象 |
| 保存方式 | 拼接参数一次提交 | 修改 `goods` 对象的字段再 `updateComGoods` |
| 商品图片 | 不支持在添加页上传 | 支持上传主图/顶部图/介绍视频 |
| 子商品图片 | 不支持 | 支持（`updateCgSetItemWithFile`） |
| 删除 | 无 | 支持删除商品/属性/子商品 |
| 售罄/下架 | 无 | 支持 toggle 切换 |
| 会员卡 | 仅在添加时传入 cardId | 可跳转选择 → 绑定 → 删除 |
| 首页广告 | 无 | 支持开启/关闭/限量份数 |
| 商品说明 | 不支持 | 支持 textarea |
| 净菜重量 | 不支持 | 支持毛重/净重/单价 |
| 子商品关联 | 不支持 | 支持关联已有商品 + 解除关联 |

## 七、注意事项

1. **添加页**保存时使用 `comSaveComGoods`，是一次性提交所有数据（包括子商品和属性列表）
2. **编辑页**的保存逻辑分多层：基本信息走 `updateComGoods`，子商品/属性各自有独立的 CRUD 接口
3. 子商品的 "允许替换" 用 `checkbox` 控制，`nxCgsiItemType` 值：0=不允许，1=允许
4. 套餐属性的子商品份数校验是前端做的（`>` 号比较，不允许超过属性的 `nxCgspLimitAmount`）
5. 划线价保存时有复杂的前端校验逻辑（`saveEditGoods` 方法中），包含了会员卡和非会员卡两种场景
6. "今日售罄" 值用的是 `-1`（售罄）和 `0`（正常），不是传统的 `0/1`
7. 分类体系是三级：爷级 (`grandId`) → 父级 (`fatherId`) → 商品，传入添加/编辑页时使用这些参数确定商品层级位置
