# 优惠券功能说明文档

> 本文档用于指导优惠券相关的增删改查。**规则券唯一主模型**（2026-06 起），不再使用「优惠券当商品」模式。

---

## ⚠️ 已废弃接口与前端约束（必读）

| 项 | 状态 | 说明 |
|----|------|------|
| `POST /nxcommunitygoods/comSaveComCouponGoods` | **已删除** | 不再创建 `sell_type=3` 券商品，不再写 `nx_community_goods` |
| `POST /nxcommunityorderssub/saveSubOrderCoupon` | **已删除** | 不再把优惠券作为订单子单/购物车商品 |
| 小程序购物车加券 | **禁止** | 不得在购物车加入优惠券商品 |
| 正确用券方式 | **结算页选券** | 用户券列表 → 结算页选择 → `order/applyCoupon`（下轮接入 Eligibility） |
| 正确创建方式 | **规则券接口** | 使用 `POST /api/nxcommunitycoupon/saveRuleCoupon`（下轮实现，字段见 `upgrade_coupon_rule_v1.sql`） |

金额主权字段：**仅 `discount_amount`**（模板表 `nx_community_coupon.discount_amount`）。

## 二、API 接口函数

所有 API 函数位于 `lib/apiCommunity.js`（除 `updateCouponWithFile` 在 `lib/apiCustomer.js`）。

### 2.1 优惠券列表

```js
import { comGetConponList } from './lib/apiCommunity.js'

// 请求参数
{
  commId: 社区ID,      // 从 wx.getStorageSync('communityInfo') 获取
  status: 0            // 0 = 获取全部
}

// 返回数据 res.result.data: Array
// 每个元素字段见下方「字段说明」
```

**请求 URL**: `POST /nxcommunitycoupon/comGetConponList`

---

### 2.2 获取优惠券详情

```js
import { comGetCouponDetail } from './lib/apiCommunity.js'

comGetCouponDetail(couponId)  // couponId = nxCommunityCouponId
```

**请求 URL**: `GET /nxcommunitycoupon/comGetCouponDetail/{id}`

---

### 2.3 创建规则券（替代已删除的 comSaveComCouponGoods）

> **`comSaveComCouponGoods` 已删除。** 不得再通过商品接口创建优惠券。

下轮将提供：

**请求 URL（计划）**: `POST /api/nxcommunitycoupon/saveRuleCoupon`

**请求 body 核心字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `nxCommunityCouponName` | String | 是 | 券名称 |
| `couponType` | String | 是 | `CASH` / `FULL_REDUCTION` |
| `discountAmount` | Number | 是 | 减免金额 |
| `thresholdAmount` | Number | 否 | 满减门槛（CASH 为 0） |
| `scopeType` | String | 是 | `ALL` / `CATEGORY` / `GOODS` |
| `scopeRefIds` | String | 条件 | JSON 数组，分类或商品 ID |
| `useChannel` | String | 否 | 默认 `ALL` |
| `nxCpCommunityId` | Number | 是 | 门店 ID |
| `couponStartDate` / `couponStopDate` 等 | String | 是 | 有效期 |

---

### 2.4 更新优惠券

```js
import { comUpdateCoupon } from './lib/apiCommunity.js'

comUpdateCoupon(data)  // data 为完整的优惠券对象
```

**请求 URL**: `POST /nxcommunitycoupon/comUpdateCoupon`

> 更新时需要传完整的优惠券对象，字段与详情返回结构一致。

---

### 2.5 删除优惠券

```js
import { delComCoupon } from './lib/apiCommunity.js'

delComCoupon({ id: couponId })
```

**请求 URL**: `POST /nxcommunitycoupon/delComCoupon`  
**请求体**: `{ id: 优惠券ID }`

| `nxCpStartDate` | String | 开始日期 |
| `nxCpStartTime` | String | 开始时间 |
| `nxCpStopDate` | String | 结束日期 |

| `nxCpStatus` | Number | 状态：`0`=启用/显示，`1`=禁用/隐藏 |
| `nxCpDownCount` | Number | 下载/领取人数 |

| `nxCpValidityType` | String | 有效期类型 |

| `nxCpFilePath` | String | 优惠券图片路径 |
| `nxCgGoodsType` | Number | 售卖类型 |

### 3.2 售卖类型（`nxCgGoodsType`）

| 值 | 含义 |
|----|------|

| `1` | 多拼 |
| `2` | 套餐 |

### 3.3 券用途（`nxCpBizPurpose`）

| 值 | 中文名称 |
|----|----------|
| `marketing` | 普通活动券 |

| `new_user_gift` | 新用户礼包券 |
| `compensation` | 补偿券 |

### 3.4 领取方式（`nxCpClaimStrategy`）

| 值 | 中文名称 |
|----|----------|
| `public_active` | 活动页主动领取 |

| `auto_grant` | 系统自动发放 |

### 3.5 有效期类型（`nxCpValidityType`）

| 值 | 中文名称 |
|----|----------|

### 3.6 状态（`nxCpStatus`）

| 值 | 含义 |
|----|------|
| `0` | 启用/显示 |
| `1` | 禁用/隐藏 |

## 五、操作流程

### 5.1 创建规则券流程

1. 从 `wx.getStorageSync('communityInfo')` 获取 `commId`
2. 构造规则券数据（`couponType`、`discountAmount`、`scopeType` 等，**不要**创建商品）
3. 调用 `saveRuleCoupon`（下轮实现）保存
4. 可选上传券图片（`updateCouponWithFile`）

### 5.1.1 小程序用券流程（禁止购物车加券）

1. 用户领券 → 我的优惠券列表
2. 下单加购**真实商品** only
3. 结算页选择用户券 → 调用 `order/applyCoupon`
4. **禁止**调用已删除的 `saveSubOrderCoupon`

### 5.2 编辑优惠券流程

1. 通过 `comGetCouponDetail(id)` 获取详情
2. 修改 `goods` 对象中的字段
3. 调用 `comUpdateCoupon(goods)` 保存更新
4. 如需修改子商品/属性，使用对应的 save/update/delete API

### 5.3 删除优惠券流程

1. 调用 `delComCoupon({ id: couponId })`
2. 成功后返回上一页

### 5.4 启用/禁用优惠券

修改 `nxCpStatus` 字段后调用 `comUpdateCoupon(item)`：
- `0` = 启用
- `1` = 禁用

## 六、售卖类型详解

### 6.1 普通（type=0）
最简单的优惠券形式，只需填写基本字段即可。

### 6.2 多拼（type=1）
需要额外填写：
- `nxCgSetSubNumber`：多拼总份数
- `nxCommunityGoodsSetItemEntities`：子商品数组，每个子商品包含名称、份数、价格

### 6.3 套餐（type=2）
需要额外填写：
- `nxCommunityGoodsSetPropertyEntities`：属性数组（如"辣度"、"冰量"）
- 每个属性下可挂 `nxCommunityGoodsSetItemEntities`：属性对应的子商品

## 七、页面导航关系

```
couponList (列表页)
  ├── → addCouponGoods (带 id=-1 进入添加模式)
  ├── → addCouponGoods (带 id=couponId 进入编辑模式)
  └── → date (日期选择辅助页)
```

- 列表页获取数据参数：`{ commId, status: 0 }`
- 进入添加模式：`wx.navigateTo({ url: '../addCouponGoods/addCouponGoods?id=-1' })`
- 进入编辑模式：`wx.navigateTo({ url: '../addCouponGoods/addCouponGoods?id=' + nxCommunityCouponId })`

## 八、MCP 操作注意事项

1. **社区信息**：所有操作前需要从 `wx.getStorageSync('communityInfo')` 获取 `commId`
2. **图片路径**：返回的图片路径是相对路径，需要拼接 `config.js` 中的 `server` 前缀
3. **日期时间格式**：日期为 `YYYY-MM-DD`，时间为 `HH:mm`（秒会补 `:00`）
4. **多拼/套餐删除**：删除属性或子商品后需要重新获取详情以刷新数据
5. **保存校验**：名称、规格、会员价为必填项
6. **售卖类型切换**：修改售卖类型时，多拼（type=1）需要先设置份数再添加子商品；套餐（type=2）需要先添加属性再添加子商品
