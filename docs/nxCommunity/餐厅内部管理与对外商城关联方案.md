# 餐厅内部管理（aigrainDesk）与对外商城（NxCommunity）关联方案

> **文档状态**：初稿梳理（2026-06-23）  
> **用途**：记录两套小程序/后端系统的业务边界、组织关联方式，以及后续「菜品关联 + AI 生成营销内容」的建设方向。  
> **后续**：本文档为活文档，可在各章节下继续补充细节、接口设计、字段映射与实施计划。

---

## 一、背景与目标

### 1.1 两套系统的定位

| 系统 | 项目目录 | 面向对象 | 核心能力 | 是否对外销售 |
|------|----------|----------|----------|:------------:|
| **餐厅配料/内部管理** | `aigrainDesk` | 老板、采购、库管、门店经营人员 | 原料采购、库存、菜品配方、成本毛利、销量分析、AI 对话经营顾问 | 否（内部经营） |
| **社区线上商城** | `nongxinle-master`（NxCommunity 模块） | 社区居民 / C 端顾客 | 像麦当劳小程序：菜品、套餐、优惠券、会员卡、首页广告、下单配送 | 是（对外销售） |

**核心诉求**：餐厅内部管理的**菜品**（`gb_distributer_food` 体系）需要与对外商城的**商品**（`nx_community_goods` 体系）建立关联，使内部经营数据能驱动外部商城的定价、促销与营销配置。

### 1.2 为什么要关联

- **aigrainDesk** 掌握真实的采购成本、配料结构、菜品销量、毛利等经营事实，但**没有对外销售渠道**。
- **NxCommunity** 是对外销售的线上商城（自提/外卖、套餐、券、会员），但**不掌握**后厨成本与销量深层分析。
- 通过组织级关联 + 菜品级映射，可以让 AI 在内部对话中分析「采购成本 + 菜品销量」后，**自动生成或建议** NxCommunity 侧可用的：
  - 商品（`NxCommunityGoods`）
  - 会员卡（`NxCommunityCard`）
  - 优惠券（`NxCommunityCoupon`）
  - 首页广告（`NxCommunityAdsense` / 商品级广告字段）

### 1.3 阅读范围约定（节省 token）

后续设计与开发时，**优先只看以下前缀的业务代码**：

| 前缀 | 含义 |
|------|------|
| `NxCommunity*` | 社区商城（门店、商品、订单、券、卡、广告等） |
| `NxCustomer*` | C 端顾客（用户、用户券、推广奖励等） |
| `NxECommerce*` | 商业组织与 GbDistributer 的桥接 |

其他无关模块（如 NxDepartment、NxPlatform、Route-Dispatch 等）除非明确涉及，否则不展开。

## 二、组织与门店层级模型

### 2.1 概念对照

```
GbDistributer（集团/配送商/组织主体）
    └── gb_department（门店、库房、厨房等组织树）
            └── gb_distributer_food（总部/门店菜品树 + 配料）

NxECommerce（商业组织，连锁店品牌层）
    └── nx_e_commerce_gb_dis_id  ──→  GbDistributer.gb_distributer_id   【已有桥接字段】
    └── NxECommerceCommunity（商业组织 ↔ 门店 多对多）
            └── NxCommunity（Community = 一个饭馆/门店）
                    └── NxCommunityGoods（对外售卖商品）
                    └── NxCommunityCoupon / Card / Adsense ...
```

**术语说明**（按业务口语）：

- **Community** = 一个饭馆 / 一个店（对外商城的门店维度）
- **ECommerce（Commercial）** = 商业组织（连锁品牌层；一个品牌下可挂多个 Community 门店）
- **GbDistributer** = 餐厅内部管理系统的组织主体（`distributerId`），与 ECommerce 通过 `nxECommerceGbDisId` 关联

### 2.2 已有组织桥接（代码现状）

**实体**：`NxECommerceEntity`

| 字段 | 说明 |
|------|------|
| `nxECommerceId` | 商业组织 ID |
| `nxECommerceGbDisId` | 关联的 GbDistributer ID（**两套系统的业务纽带**） |
| `nxECommerceName` / `nxECommerceImg` | 品牌名称与图片 |

**中间表**：`NxECommerceCommunityEntity`（`nx_e_commerce_community`）

| 字段 | 说明 |
|------|------|
| `nxEccEId` | 商业组织 ID |
| `nxEccCommunityId` | 门店（Community）ID |

**门店**：`NxCommunityEntity`

| 字段 | 说明 |
|------|------|
| `nxCommunityId` | 门店 ID |
| `nxCommunityCommerceId` | 所属商业组织 ID |
| `nxCommunityName`、经纬度、营业时间、配送规则等 | 对外商城门店信息 |

**已有 API**（`NxECommerceController`）：

| 接口 | 用途 |
|------|------|
| `GET api/nxecommerce/gbGetECommerce/{gbDisId}` | 由 GbDistributer ID 查商业组织 |
| `GET api/nxecommerce/getGbCommunity/{eCommerceId}` | 由商业组织查下属 Community 列表 |
| `POST api/nxecommerce/registerGbCommerce` | 注册商业组织并绑定 GbDis |

> **结论**：组织级关联**已有基础**；**菜品 ↔ 商品** 的细粒度关联尚待设计与实现（见 §五）。

## 三、两侧「菜品/商品」模型对照

### 3.1 餐厅内部：GbDistributerFood（aigrainDesk）

**表/实体**：`gb_distributer_food` → `GbDistributerFoodEntity`

| 关键字段 | 说明 |
|----------|------|
| `gbDistributerFoodId` | 菜品 ID |
| `gbDfDistributerId` | 所属 GbDistributer |
| `gbDfFoodName` / `gbDfFoodPrice` | 菜品名称、标价 |
| `gbDfFoodFatherId` | 父级（分类树） |
| `gbDfNxFoodId` | 与 nx 标准菜库的关联（如有） |
| `gbdisFoodGoodsEntities` | 配料 / 原料关联（`GbDistributerFoodGoodsEntity`） |

**AI 侧已有能力**（`aigrainDesk/src/main/java/com/nongxinle/ai`）：

- 菜品毛利分析（DishProfit / D-7）
- 菜品销量排行（DishSales / D-8）
- 菜单经营顾问（MenuOperationAgent）：综合毛利、销量、定价建议
- 采购成本、库存、出库等域可组合进 BusinessDiagnosis

**AI 入口示例**：`GET /api/ai/advisors/{advisorId}?scene=MINIAPP`（如 advisorId=13）

### 3.2 对外商城：NxCommunityGoods（nongxinle-master）

**表/实体**：`nx_community_goods` → `NxCommunityGoodsEntity`

详见：[菜品业务模型与字段参考.md](./菜品业务模型与字段参考.md)

**售卖类型**（`nxCgGoodsType`）：

| 值 | 名称 | 说明 |
|:--:|------|------|
| 0 | 普通 | 单菜品直接售卖 |
| 1 | 自由拼 | 顾客自选子商品，按单品累加计价 |
| 2 | 套餐 | 按属性分组（主食/饮品/小菜） |
| 3 | 多拼 | 固定一口价，自由选子商品 |

**与内部经营相关的已有字段**（商品表内，尚未明确指向 GbFood）：

| 字段 | 说明 |
|------|------|
| `nxCgDistributerId` | 批发商/配送商 ID（历史字段，可能与 GbDis 有关） |
| `nxCgDistributerGoodsId` | 批发商商品 ID |
| `nxCgBuyingPrice` | 采购价 |
| `nxCgExpectGrossProfit` / `nxCgRealityGrossProfit` | 预期/实际毛利 |

**子结构**：

- `NxCommunityGoodsSetPropertyEntity` — 套餐属性
- `NxCommunityGoodsSetItemEntity` — 子商品（可 `nxCgsiItemCgGoodsId` 关联已有商品）
- `NxCommunityFatherGoodsEntity` — 三级分类（爷级 → 父级 → 商品）

### 3.3 两侧模型差异摘要

| 维度 | aigrainDesk（内部） | NxCommunity（对外） |
|------|---------------------|---------------------|
| 核心实体 | `GbDistributerFood` + 配料 | `NxCommunityGoods` + SetItem/Property |
| 价格逻辑 | 成本 + 标价 + 毛利分析 | 会员价、划线价、满减、券后价 |
| 售卖形态 | 部门菜、总部菜树 | 普通/自由拼/套餐/多拼 |
| 营销 | 无 C 端券/卡/广告 | Coupon、Card、Adsense |
| 组织锚点 | `gbDfDistributerId` | `nxCgCommunityId` + `nxCgCommerceId` |

## 四、NxCommunity 营销与 C 端实体

### 4.1 会员卡 — `NxCommunityCardEntity`

| 字段 | 说明 |
|------|------|
| `nxCommunityCardId` / `nxCommunityCardName` | 卡 ID、名称 |
| `nxCcPrice` / `nxCcWords` | 价格、说明 |
| `nxCcCommunityId` | 所属门店 |
| `nxCommunityGoodsEntityList` | 绑定商品列表（商品侧 `nxCgCardId` 反向关联） |

### 4.2 优惠券 — `NxCommunityCouponEntity`

> 2026-06 起为**规则券**主模型，不再把券当商品卖。详见 [优惠券功能说明.md](./优惠券功能说明.md)

| 字段 | 说明 |
|------|------|
| `couponType` | `CASH` / `FULL_REDUCTION` |
| `discountAmount` / `thresholdAmount` | 减免金额 / 满减门槛 |
| `scopeType` / `scopeRefIds` | 适用范围：`ALL` / `CATEGORY` / `GOODS` + JSON ID 数组 |
| `useChannel` | `ALL` / `POS` / `MINIAPP` |
| `nxCpBizPurpose` | `marketing` / `referral_register` / `new_user_gift` / `compensation` |
| `nxCpClaimStrategy` | `public_active` / `reward_only` / `auto_grant` |
| `nxCpCommunityId` | 所属门店 |

**C 端用券**：结算页选券 → `order/applyCoupon`；**禁止**购物车加券商品。

### 4.3 首页广告

**独立广告位** — `NxCommunityAdsenseEntity`：

| 字段 | 说明 |
|------|------|
| `nxCaCommunityId` | 门店 |
| `nxCaCgGoodsId` | 跳转关联商品 |
| `nxCaFilePath` / `nxCaClickTo` | 图片、链接 |
| 时间、排序、状态 | 投放控制 |

**商品级广告**（字段在 `NxCommunityGoodsEntity` 上）：

- `nxCgIsOpenAdsense`、`nxCgAdsenseStartTime/StopTime`
- `nxCgAdsenseStockQuantity` / `nxCgAdsenseRestQuantity`

### 4.4 C 端顾客 — `NxCustomer*`

| 模块 | 说明 |
|------|------|
| `NxCustomerUser*` | 小程序用户 |
| `NxCustomerUserCouponEntity` | 用户持有的券实例（关联 `nxCucCouponId`、`nxCucCommunityId`） |
| 推广奖励 | 见 [NxCustomerUser推广注册奖励接口说明.md](./NxCustomerUser推广注册奖励接口说明.md) |

## 五、待建设：菜品 ↔ 商品关联（核心缺口）

### 5.1 当前状态

- ✅ **组织级**：`NxECommerce.nxECommerceGbDisId` ↔ `GbDistributer`
- ✅ **门店级**：`NxECommerceCommunity` ↔ `NxCommunity`
- ❓ **菜品级**：`GbDistributerFood` ↔ `NxCommunityGoods` **尚无统一外键字段**（需设计）

`NxCommunityGoods` 上虽有 `nxCgDistributerId`、`nxCgDistributerGoodsId`，语义偏向「批发商商品/采购」，**不等同于** `gbDistributerFoodId`。

### 5.2 关联设计方向（待确认）

以下为建议方向，**尚未定稿**，后续章节补充：

```
GbDistributerFood.gb_distributer_food_id
        ↕ （待新增映射表或商品表字段，如 nx_cg_gb_food_id）
NxCommunityGoods.nx_community_goods_id
```

**映射需携带的上下文**：

- `gbDfDistributerId` → 经 `NxECommerce` 找到 `nxCgCommerceId`
- 具体门店：`gb_department`（门店）↔ `nxCommunityId`（若一 Dis 多 Community，需部门级映射规则）
- 同步策略：单向推送（内部 → 商城）还是双向编辑权分离

### 5.3 同步内容范围（初稿）

| 内部字段/分析结果 | 可同步到 NxCommunity |
|-------------------|----------------------|
| 菜品名称、图片、说明 | `nxCgGoodsName`、图片路径、`nxCgGoodsDetail` |
| 标价 / AI 建议价 | `nxCgGoodsPrice`、划线价 `nxCgGoodsHuaxianPrice` |
| 套餐结构 | `nxCgGoodsType=2` + SetProperty + SetItem |
| 成本与毛利结论 | `nxCgBuyingPrice`、促销字段、满减规则 |
| AI 营销建议 | 生成 `NxCommunityCoupon`、`NxCommunityCard`、Adsense 配置 |

## 六、AI 生成 NxCommunity 营销内容的设想流程

### 6.1 场景描述

在 **aigrainDesk 小程序端**，老板通过 **AI 对话** 讨论：

- 近期采购成本变化
- 哪些菜卖得好 / 卖得差
- 哪些菜毛利低但在爆卖
- 是否该涨价、做促销、推套餐

AI（MenuOperationAgent + DishProfit + DishSales 等）给出经营建议后，**可选动作**：一键或确认后生成/更新 NxCommunity 侧配置。

### 6.2 目标产出物

| 产出 | 实体 | 典型触发场景 |
|------|------|--------------|
| 调价 / 划线价促销 | `NxCommunityGoods` 促销字段 | 高销量低毛利 → 涨价或限时划线价 |
| 满减活动 | 商品 `nxCgPromotionType=2` 或规则券 | 拉动滞销菜、组合销售 |
| 优惠券 | `NxCommunityCouponEntity` | 新客礼包、补偿券、活动券 |
| 会员卡 | `NxCommunityCardEntity` | 绑定高频复购菜品 |
| 首页广告 | `NxCommunityAdsense` 或商品 Adsense 字段 | 推爆品、限时限量 |

### 6.3 前置条件（实施 AI 写商城前必须满足）

1. 组织桥接已配置：`registerGbCommerce` + ECommerceCommunity 绑定门店
2. 菜品 ↔ 商品映射已建立（§5.2）
3. AI 侧能解析当前会话的 `distributerId` / 门店 scope，并反查对应 `nxCommunityId`
4. 写接口权限：aigrainDesk 调用 nongxinle-master 的 NxCommunity API（或共用 DB 的 Service 层）

### 6.4 流程草图

```
老板在 aigrainDesk AI 对话
    → 分析采购成本 + 菜品销量/毛利（已有 Tool / AnswerPlan）
    → 输出经营建议 + 「是否同步到对外商城？」
    → 用户确认
    → 按映射关系写入 NxCommunityGoods / Coupon / Card / Adsense
    → 居民小程序（NxCustomer）可见新价格/券/广告
```
