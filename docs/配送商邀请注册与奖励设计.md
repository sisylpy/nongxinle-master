# 配送商邀请注册与奖励设计

## 一、业务场景

1. **A 邀请 B 成为协作伙伴**，B 可能尚未注册
2. **B 通过 A 的邀请链接/邀请码完成注册**
3. **记录 B 是否注册成功**，用于给 A（宣传方）发放奖励

---

## 二、数据表设计

### 1. nx_distributer 表增加字段

```sql
-- 邀请我注册的配送商 id（谁邀请的我）
ALTER TABLE nx_distributer ADD COLUMN invited_by_nx_distributer_id INT DEFAULT NULL COMMENT '邀请我注册的配送商id';

-- 我的邀请码（6-8位，用于生成邀请链接，供别人注册时使用）
ALTER TABLE nx_distributer ADD COLUMN invite_code VARCHAR(16) DEFAULT NULL COMMENT '我的邀请码' UNIQUE;
```

### 2. 新建 nx_distributer_invite 表（邀请记录）

```sql
CREATE TABLE nx_distributer_invite (
  nx_distributer_invite_id INT PRIMARY KEY AUTO_INCREMENT,
  inviter_nx_distributer_id INT NOT NULL COMMENT '邀请人/宣传人配送商id',
  invite_code VARCHAR(16) NOT NULL COMMENT '邀请码',
  invitee_phone VARCHAR(20) DEFAULT NULL COMMENT '被邀请人手机号（预填）',
  invite_type TINYINT DEFAULT 1 COMMENT '1=邀请成为我的供货商,2=邀请购买我的产品',
  status TINYINT DEFAULT 0 COMMENT '0=待注册,1=已注册,2=已过期',
  invitee_nx_distributer_id INT DEFAULT NULL COMMENT '被邀请人注册成功后的配送商id',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  registered_at DATETIME DEFAULT NULL COMMENT '被邀请人注册成功时间',
  reward_status TINYINT DEFAULT 0 COMMENT '0=待发放,1=已发放',
  reward_amount DECIMAL(10,2) DEFAULT NULL COMMENT '奖励金额',
  reward_type VARCHAR(32) DEFAULT NULL COMMENT '奖励类型：points=试用点数,cash=现金等',
  UNIQUE KEY uk_invite_code (invite_code),
  KEY idx_inviter (inviter_nx_distributer_id),
  KEY idx_status (status),
  KEY idx_reward (reward_status)
) COMMENT '配送商邀请注册记录表';
```

## 三、业务流程

### 流程 1：A 邀请未注册的 B

```
A（已注册配送商） → 生成邀请链接/邀请码
  ↓
  调用接口：创建邀请记录
  - inviter_nx_distributer_id = A
  - invite_code = 唯一码（可用 A 的 invite_code 或新生成）
  - invite_type = 1 或 2
  - status = 0（待注册）
  ↓
  返回：邀请链接（含 invite_code）、二维码
```

### 流程 2：B 通过邀请注册

```
B 打开邀请链接 / 扫码
  ↓
  注册页预填：invite_code（从链接带过来）
  ↓
  B 完成注册（调用现有 disAndUserSave 或新接口）
  ↓
  注册成功后：
  1. nx_distributer.invited_by_nx_distributer_id = A
  2. nx_distributer.invite_code = 为 B 生成其专属邀请码
  3. 更新 nx_distributer_invite：status=1, invitee_nx_distributer_id=B, registered_at=now
  4. 建立 A-B 协作关系（nx_distributer_nx_distributer）
  5. 若 invite_type=1，创建 block（不给他看我的商品）
  6. reward_status=0（待发放）
```

### 流程 3：奖励发放

```
后台/定时任务 扫描 reward_status=0 且 status=1 的记录
  ↓
  按规则计算奖励（如：每成功邀请 1 人 = 10 试用点）
  ↓
  发放给 inviter（增加 nx_distributer 的试用点数等）
  ↓
  更新 reward_status=1, reward_amount, reward_type
```

## 四、接口设计

### 1. 创建邀请（A 生成邀请链接）

**路径：** `POST /api/nxdistributernxdistributer/createPartnerInvite`

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inviterDisId | Integer | 是 | 邀请人配送商 id |
| inviteType | Integer | 是 | 1=邀请成为供货商，2=邀请购买产品 |
| inviteePhone | String | 否 | 被邀请人手机号（预填，可选） |

**返回：**

```json
{
  "code": 0,
  "data": {
    "inviteCode": "ABC123",
    "inviteUrl": "pages/register/register?inviteCode=ABC123",
    "expireDays": 7
  }
}
```

### 2. 校验邀请码（B 注册前）

**路径：** `POST /api/nxdistributernxdistributer/validateInviteCode`

**参数：** inviteCode

**返回：** 邀请人简要信息、inviteType，用于注册页展示「您正在通过 XX 的邀请注册」

### 3. 配送商注册（改造 disAndUserSave）

**路径：** `POST /api/nxdistributer/disAndUserSave`

**请求体**：在原有 `NxDistributerEntity` 中增加可选字段 `inviteCode`（邀请链接中的邀请码）

- 若带 inviteCode：校验有效 → 注册成功后自动建立协作、更新 invite 记录、设置 invited_by、生成新用户专属 invite_code

### 4. 邀请记录列表（A 查看我发出的邀请）

**路径：** `POST /api/nxdistributernxdistributer/myInviteList`

**参数：** inviterDisId

**返回：** 邀请列表，含 status、registered_at、reward_status

### 5. 待发放奖励列表（后台用）

**路径：** `POST /api/nxdistributernxdistributer/pendingRewardList`

**返回：** reward_status=0 且 status=1 的记录

### 6. 发放奖励（后台/定时）

**路径：** `POST /api/nxdistributernxdistributer/grantReward`

**参数：** inviteId

**逻辑：** 给 inviter 增加试用点数，更新 reward_status=1

## 五、奖励规则建议

| 奖励类型 | 说明 | 实现方式 |
|----------|------|----------|
| 试用点数 | 每成功邀请 1 人赠送 N 点 | 增加 inviter 的 nx_distributer_buy_quantity |
| 阶梯奖励 | 邀请 5 人送 X，10 人送 Y | 按累计成功数计算 |
| 首单奖励 | 被邀请人首单后奖励 | 需关联订单，较复杂 |

**建议**：先实现「每成功邀请 1 人 = 固定试用点数」，与现有 `sysCmRegisterGiftPoints` 类似，可配置为 `sysCmInviteRewardPoints`。

## 六、实现优先级

| 优先级 | 内容 |
|--------|------|
| P0 | nx_distributer 增加 invited_by、invite_code；nx_distributer_invite 表 |
| P0 | createPartnerInvite、validateInviteCode |
| P0 | 改造注册接口支持 inviteCode，注册成功后建协作+更新 invite |
| P1 | myInviteList、pendingRewardList、grantReward |
| P2 | 奖励规则配置、阶梯奖励 |

## 七、与现有协作的关系

| 场景 | 处理方式 |
|------|----------|
| B 已注册 | 走现有 saveBusiness，不创建 invite 记录 |
| B 未注册 | A 发邀请 → B 注册时带 inviteCode → 注册成功后自动建协作 + 创建 invite 记录 |
