# 优果团助手 — 产品开发计划

> 版本：v1.0  
> 日期：2026-06-24  
> 定位：水果生鲜门店社区团购团长群报单统计 SaaS 工具

---

## 一、产品定位与核心价值

### 1.1 目标用户

| 用户角色 | 痛点 | 产品价值 |
|---------|------|---------|
| 门店团长 | 群消息99+，人工统计错漏多 | 自动识别汇总，零错漏 |
| 门店老板 | 采购无数据支撑，盲目进货 | 多维度数据统计，精准采购 |
| 连锁总部 | 多店多群不同步，管理混乱 | 一键开团，分店实时同步 |
| 顾客 | 手写核销慢，体验差 | 扫码核销，状态清晰可查 |

### 1.2 核心差异化

- **企微合规**：基于企业微信架构，不爬微信个人号，无封号风险
- **零安装**：企微内直接使用，团长顾客均无门槛
- **AI 自动化**：订单识别 + 开团文案生成，降低运营工作量

## 二、核心功能模块

### 2.1 模块总览

```
优果团助手
├── 订单自动统计模块
├── 核销管理模块
├── AI 开团模块
├── 连锁多店管理模块
├── 会员管理模块
├── 数据报表模块
└── 企微集成模块
```

### 2.2 功能详细说明

#### M1：订单自动统计
- **群消息监听**：通过企微消息回调 API 获取群内报单消息
- **AI 订单识别**：NLP 解析商品名、数量、规格、收货人
- **订单汇总表**：实时生成今日/本期订单汇总
- **异常提示**：识别失败自动标记，人工复核

#### M2：核销管理
- **核销码/二维码**：每笔订单生成唯一核销码
- **状态管理**：待取 / 已取 / 预留 三种状态
- **操作撤销**：核销错误可撤销，记录操作日志
- **取货提醒**：到货后自动群内 @ 未取货顾客

#### M3：AI 开团
- **模板管理**：预设开团/截团/到货提醒文案模板
- **一键生成**：选择商品 + 模板，AI 生成完整开团文案
- **定时发送**：支持预约发送时间
- **多群同步**：一键转发至多个门店群

#### M4：连锁多店管理
- **总部管控**：总部账号一键开团，全部门店同步
- **价格/活动同步**：总部改价，分店实时生效
- **多店切换**：团长可切换管理多个门店
- **独立结算**：各门店订单/对账独立统计

#### M5：会员管理
- **自动标签**：新客 / 老客 / 未取货 / 高价值客户自动标记
- **精准触达**：按标签筛选，定向发送营销消息
- **会员等级**：支持设置会员等级与折扣规则
- **消费记录**：客户历史订单可查

#### M6：数据报表
- **订单报表**：按日/周/月/商品维度统计
- **核销率统计**：各门店核销率对比分析
- **采购建议**：基于历史销量生成采购建议清单
- **导出功能**：支持 Excel 导出

## 三、技术架构设计

### 3.1 整体架构

```
┌─────────────────────────────────────────┐
│           企业微信前端入口                │
│  （企微应用 + 企微群机器人）              │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           API 网关层                     │
│    （认证 / 限流 / 路由）                │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           业务服务层                      │
│  ┌────────┐ ┌────────┐ ┌───────────┐  │
│  │订单服务 │ │核销服务 │ │ AI 开团   │  │
│  └────────┘ └────────┘ └───────────┘  │
│  ┌────────┐ ┌────────┐ ┌───────────┐  │
│  │门店管理 │ │会员服务 │ │ 数据报表  │  │
│  └────────┘ └────────┘ └───────────┘  │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           数据与基础设施层                 │
│  MySQL  │ Redis  │ 腾讯云 COS │ AI 服务 │
└─────────────────────────────────────────┘
```

### 3.2 技术选型

| 层级 | 技术栈 | 说明 |
|-----|--------|------|
| 前端 | 企微 H5 应用 + 微信小程序 | 企微内嵌 H5，顾客端可用小程序 |
| 后端 | Spring Boot + MyBatis-Plus | 与现有技术栈一致 |
| 数据库 | MySQL 8.0 + Redis | 主数据存储 + 缓存 |
| 消息队列 | RabbitMQ | 异步处理订单识别任务 |
| AI 能力 | 腾讯混元 / 自建 NLP 模型 | 订单解析 + 文案生成 |
| 部署 | 腾讯云服务器 + Docker | 与现有 grainservice.club 同环境 |
| 企微集成 | 企微第三方应用 SDK | 消息回调 + 身份认证 |

### 3.3 企微集成方案

**关键：合规获取群消息**

- 使用**企微第三方应用**身份，申请 `应用可见范围` 内的群消息读取权限
- 通过企微**消息回调**接收群内消息（仅限应用所在群）
- **不涉及**微信个人号，规避封号风险
- 会话存档功能需单独申请（增值功能）

## 四、开发阶段规划

### 阶段一：MVP 核心版（6-8 周）

**目标**：验证核心假设——自动统计订单 + 核销管理

| 周次 | 任务 | 交付物 |
|-----|------|--------|
| W1-2 | 企微第三方应用接入，消息回调打通 | 能接收群消息 |
| W3-4 | 订单 AI 识别引擎开发 + 订单数据模型 | 识别准确率 >85% |
| W5-6 | 核销功能开发（码生成 + 状态管理 + 撤销） | 完整核销流程 |
| W7-8 | 数据报表 V1（订单汇总 + 导出） + 内测 | MVP 可演示版本 |

### 阶段二：运营增强版（4-6 周）

**目标**：AI 开团 + 会员管理，提升运营效率

| 周次 | 任务 | 交付物 |
|-----|------|--------|
| W9-10 | AI 开团文案生成 + 模板管理 | 一键开团功能 |
| W11-12 | 会员自动标签 + 精准触达 | 会员管理后台 |
| W13-14 | 定时发送 + 多群同步转发 | 运营自动化 |

### 阶段三：连锁扩展版（4-6 周）

**目标**：多店管理 + 总部管控，支持连锁扩张

| 周次 | 任务 | 交付物 |
|-----|------|--------|
| W15-16 | 多门店架构重构（总部/分店角色） | 多店管理后台 |
| W17-18 | 总部一键开团 + 价格活动同步 | 连锁管控功能 |
| W19-20 | 独立结算 + 分店数据隔离 | 完整连锁版本 |

### 阶段四：数据智能化（持续迭代）

- 采购建议 AI 模型训练
- 核销率预测与预警
- 顾客消费行为分析
- 与现有 gbDistributer / grainservice.club 数据打通

## 七、成功指标（OKR）

### 第一阶段目标（MVP 上线后 3 个月）

- **用户指标**：接入门店 ≥ 10 家，活跃团长 ≥ 20 人
- **效率指标**：订单统计时间从 30 分钟/天 → 3 分钟/天
- **准确率**：订单识别准确率 ≥ 90%，核销错误率 ≤ 1%
- **留存率**：周活跃团长占比 ≥ 70%

### 长期目标（1 年内）

- 覆盖门店 ≥ 100 家
- 月订单处理量 ≥ 50000 笔
- 与 grainservice.club 会员体系打通
- 启动批发采购模块（gbDistributer）与团长系统联动
